# Contributing to CivicPulse

Thank you for your interest in contributing to CivicPulse! We welcome bug fixes, documentation improvements, feature enhancements, and testing contributions.

## Getting Started
1. Fork the repository on GitHub.
2. Clone your fork locally.
3. Create a new branch.
4. Make your changes.
5. Commit and push your changes.
6. Open a pull request.

### Clone Your Fork

git clone https://github.com/<your-username>/civicpulse-smart-city.git
cd civicpulse-smart-city

### Create a New Branch

git checkout -b feature/your-feature-name

## Branch Naming Conventions

Use clear and descriptive branch names.

Examples:
- docs/add-contributing-guide
- fix/login-validation
- feat/add-dashboard-widget

## Development Setup

Follow the setup instructions in README.md to configure:

- MySQL
- Spring Boot backend
- Angular frontend
- Python sentiment service

## Commit Message Conventions

Use descriptive commit messages.

Examples:
- docs: add contributing guidelines
- fix: resolve login validation bug
- feat: add sentiment analysis dashboard

## Pull Request Workflow

1. Ensure your changes are focused on a single issue.
2. Test your changes thoroughly.
3. Update documentation if needed.
4. Commit and push your branch.
5. Open a pull request against the main branch.
6. Link the related issue number.
7. Respond to review comments and make any requested changes.

### Pull Request Checklist

Include the following in your pull request:

- Summary of changes
- Related issue number
- Screenshots (if UI changes)
- Testing notes

## Coding Standards

### Backend (Java / Spring Boot)
- Follow standard Java naming conventions.
- Keep controllers, services, and repositories modular.
- Write clear and descriptive method names.

### Frontend (Angular)
- Follow the Angular Style Guide.
- Keep components focused and reusable.
- Use reactive forms and services appropriately.

### Python AI Service
- Follow PEP 8 formatting.
- Keep modules small and well-documented.

## Beginner-Friendly Contribution Tips
- Start with documentation improvements or small bug fixes.
- Read the README.md carefully before making changes.
- Keep pull requests small and focused.
- Ask questions if anything is unclear.
- Review existing code and commit patterns to stay consistent.

## Reporting Bugs
When opening a bug report, include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs (if applicable)

## Suggesting Features
Feature requests should clearly explain:

- The problem being solved
- Proposed solution
- Expected benefits

## Code of Conduct

Please be respectful and constructive in all interactions.

## Need Help?

If you have questions, open an issue or start a discussion in the repository.
Thank you for contributing to CivicPulse and helping build smarter cities through technology!
